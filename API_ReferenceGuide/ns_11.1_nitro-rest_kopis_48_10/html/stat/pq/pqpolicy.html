pqpolicy

Statistics for PQ policy resource.

Read/write properties

policyname<String>

Name of the priority queuing policy whose statistics must be displayed. If a name is not provided, statistics of all priority queuing policies are shown.
Minimum length = 1

clearstats<String>

Clear the statsistics / counters.
Possible values = basic, full

Read only properties

pqtotavgqueuewaittime<Double>

Average wait time for clients for this priority queuing policy.

pqavgqueuewaittimerate<Double>

Rate (/s) counter for pqtotavgqueuewaittime

pqavgclienttransactiontimems<Double>

Average time taken by a priority queuing client to complete its transaction for this priority queuing policy.

pqavgclienttransactiontimemsrate<Double>

Rate (/s) counter for pqavgclienttransactiontimems

pqvserverip<String>

IP address of the virtual server to which this priority queuing policy is bound.

pqvserverport<Integer>

Port number of the virtual server to which this priority queuing policy is bound.

pqqdepth<Double>

Number of clients waiting currently for this priority queuing policy.

pqqdepthrate<Double>

Rate (/s) counter for pqqdepth

pqcurrentclientconnections<Double>

Current number of server connections established for serving clients for this priority queuing policy.

pqcurrentclientconnectionsrate<Double>

Rate (/s) counter for pqcurrentclientconnections

pqtotclientconnections<Double>

Total number of server connections established for serving clients for this priority queuing policy.

pqclientconnectionsrate<Double>

Rate (/s) counter for pqtotclientconnections

pqdropped<Double>

Total number of dropped transactions for this priority queuing policy.

pqdroppedrate<Double>

Rate (/s) counter for pqdropped

totclienttransactions<Double>

Total number of client transactions for this priority queuing policy.

clienttransactionsrate<Double>

Rate (/s) counter for totclienttransactions

pqtotqueuedepth<Double>

Total number of waiting clients for this priority queuing policy.

pqqueuedepthrate<Double>

Rate (/s) counter for pqtotqueuedepth

Operations

You can perform the following operations:

get (all)get

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Fetch statistics of the bound resources: NITRO allows you to fetch statistics of bound resources by specifying the "statbindings" query parameter as "yes". This query parameter is applicable only to get(byname) operation. For example, to get binding statistics of a given lbvserver, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/stat/lbvserver/lbvip1?statbindings=yes

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
