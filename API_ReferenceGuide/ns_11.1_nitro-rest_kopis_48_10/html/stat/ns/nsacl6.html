nsacl6

Statistics for ACL6 entry resource.

Read/write properties

acl6name<String>

Name of the ACL6 rule whose statistics you want the NetScaler appliance to display.
Minimum length = 1

clearstats<String>

Clear the statsistics / counters.
Possible values = basic, full

Read only properties

acl6totpktsbridged<Double>

Packets matching a bridge IPv6 ACL, which is in transparent mode and bypasses service processing.

acl6pktsbridgedrate<Double>

Rate (/s) counter for acl6totpktsbridged

acl6totpktsdenied<Double>

Packets dropped because they match IPv6 ACLs with processing mode set to DENY.

acl6pktsdeniedrate<Double>

Rate (/s) counter for acl6totpktsdenied

acl6totpktsallowed<Double>

Packets matching IPv6 ACLs with processing mode set to ALLOW. NetScaler processes these packets.

acl6pktsallowedrate<Double>

Rate (/s) counter for acl6totpktsallowed

acl6totpktsnat<Double>

Packets matching a NAT ACL6, resulting in a NAT session.

acl6pktsnatrate<Double>

Rate (/s) counter for acl6totpktsnat

acl6tothits<Double>

Packets matching an IPv6 ACL.

acl6hitsrate<Double>

Rate (/s) counter for acl6tothits

acl6totmisses<Double>

Packets not matching any IPv6 ACL.

acl6missesrate<Double>

Rate (/s) counter for acl6totmisses

acl6totpktsnat64<Double>

Packets matching a NAT64 ACL6, resulting in a NAT64 translation.

acl6pktsnat64rate<Double>

Rate (/s) counter for acl6totpktsnat64

acl6totcount<Double>

Total number of ACL6 rules configured.

acl6perhits<Double>

Number of times the acl6 was hit

acl6perhitsrate<Double>

Rate (/s) counter for acl6perhits

Operations

You can perform the following operations:

get (all)get

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Fetch statistics of the bound resources: NITRO allows you to fetch statistics of bound resources by specifying the "statbindings" query parameter as "yes". This query parameter is applicable only to get(byname) operation. For example, to get binding statistics of a given lbvserver, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/stat/lbvserver/lbvip1?statbindings=yes

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
