dospolicy

Statistics for DoS policy resource.

Read/write properties

name<String>

The name of the DoS protection policy whose statistics must be displayed. If a name is not provided, statistics of all the DoS protection policies are displayed.
Minimum length = 1

clearstats<String>

Clear the statsistics / counters.
Possible values = basic, full

Read only properties

doscurrentcltdetectrate<Double>

Current ratio of JavaScript send rate to the server response rate (Client detect rate)

dosphysicalserviceip<String>

IP address of the service to which this policy is bound.

dosphysicalserviceport<Integer>

Port address of the service to which this policy is bound.

doscurrentqueuesize<Double>

Current queue size of the server to which this policy is bound.

doscurrentqueuesizerate<Double>

Rate (/s) counter for doscurrentqueuesize

dostotjssent<Double>

Total number of DoS JavaScript transactions performed for this policy.

dosjssentrate<Double>

Rate (/s) counter for dostotjssent

dostotjsrefused<Double>

Number of times the DoS JavaScript was not sent because the set JavaScript rate was not met for this policy.

dosjsrefusedrate<Double>

Rate (/s) counter for dostotjsrefused

dostotvalidclients<Double>

Total number of valid DoS cookies received for this policy.

dosvalidclientsrate<Double>

Rate (/s) counter for dostotvalidclients

dostotjsbytessent<Double>

Total number of DoS JavaScript bytes sent for this policy.

dosjsbytessentrate<Double>

Rate (/s) counter for dostotjsbytessent

dostotnongetpostrequests<Double>

Number of non-GET and non-POST requests for which DOS JavaScript was sent.

dosnongetpostrequestsrate<Double>

Rate (/s) counter for dostotnongetpostrequests

Operations

You can perform the following operations:

get (all)get

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Fetch statistics of the bound resources: NITRO allows you to fetch statistics of bound resources by specifying the "statbindings" query parameter as "yes". This query parameter is applicable only to get(byname) operation. For example, to get binding statistics of a given lbvserver, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/stat/lbvserver/lbvip1?statbindings=yes

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
