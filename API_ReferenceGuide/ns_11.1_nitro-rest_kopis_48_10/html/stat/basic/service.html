service

Statistics for service resource.

Read/write properties

name<String>

Name of the service.
Minimum length = 1

clearstats<String>

Clear the statsistics / counters.
Possible values = basic, full

Read only properties

throughput<Double>

Number of bytes received or sent by this service (Mbps).

throughputrate<Double>

Rate (/s) counter for throughput

avgsvrttfb<Double>

Average TTFB between the NetScaler appliance and the server.TTFB is the time interval between sending the request packet to a service and receiving the first response from the service

primaryipaddress<String>

The IP address on which the service is running.

primaryport<Integer>

The port on which the service is running.

servicetype<String>

The service type of this service.Possible values are ADNS, DNS, MYSQL, RTSP, SSL_DIAMETER, ADNS_TCP, DNS_TCP, NNTP, SIP_UDP, SSL_TCP, ANY, FTP, RADIUS, SNMP, TCP, DHCPRA, HTTP, RDP, SSL, TFTP, DIAMETER, MSSQL, RPCSVR, SSL_BRIDGE, UDP

state<String>

Current state of the server. Possible values are UP, DOWN, UNKNOWN, OFS(Out of Service), TROFS(Transition Out of Service), TROFS_DOWN(Down When going Out of Service)

totalrequests<Double>

Total number of requests received on this service or virtual server. (This applies to HTTP/SSL services and servers.)

requestsrate<Double>

Rate (/s) counter for totalrequests

totalresponses<Double>

Number of responses received on this service or virtual server. (This applies to HTTP/SSL services and servers.)

responsesrate<Double>

Rate (/s) counter for totalresponses

totalrequestbytes<Double>

Total number of request bytes received on this service or virtual server.

requestbytesrate<Double>

Rate (/s) counter for totalrequestbytes

totalresponsebytes<Double>

Number of response bytes received by this service or virtual server.

responsebytesrate<Double>

Rate (/s) counter for totalresponsebytes

curclntconnections<Double>

Number of current client connections.

surgecount<Double>

Number of requests in the surge queue.

cursrvrconnections<Double>

Number of current connections to the actual servers behind the virtual server.

svrestablishedconn<Double>

Number of server connections in ESTABLISHED state.

curreusepool<Double>

Number of requests in the idle queue/reuse pool.

maxclients<Double>

Maximum open connections allowed on this service.

curload<Double>

Load on the service that is calculated from the bound load based monitor.

curtflags<Double>

Current flags on the service for internal use in display handlers.

vsvrservicehits<Double>

Number of times that the service has been provided.

vsvrservicehitsrate<Double>

Rate (/s) counter for vsvrservicehits

activetransactions<Double>

Number of active transactions handled by this service. (Including those in the surge queue.) Active Transaction means number of transactions currently served by the server including those waiting in the SurgeQ

Operations

You can perform the following operations:

get (all)get

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Fetch statistics of the bound resources: NITRO allows you to fetch statistics of bound resources by specifying the "statbindings" query parameter as "yes". This query parameter is applicable only to get(byname) operation. For example, to get binding statistics of a given lbvserver, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/stat/lbvserver/lbvip1?statbindings=yes

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
