SNMP Get-Response PDUs that have been generated by the NetScaler.

snmpresponsesrate<Double>

Rate (/s) counter for snmptotresponses

snmptottraps<Double>

SNMP Trap PDUs that have been generated by the NetScaler.

snmptoterrreqdropped<Double>

SNMP requests dropped.

snmptotparseerrs<Double>

Number of ASN.1 or BER errors encountered when decoding received SNMP Messages.

snmptotbadversions<Double>

Number of SNMP messages received, which were for an unsupported SNMP version.

snmptotbadcommname<Double>

SNMP messages received, which used an SNMP community name not known to the NetScaler.

snmptotbadcommuse<Double>

The total number of SNMP Messages received that represented an SNMP operation which was not allowed by the SNMP community named in the Message.

snmpunsupportedsecuritylevel<Double>

SNMP packets that were dropped because they requested a security level that was unknown to the NetScaler or otherwise unavailable.

snmpnotintimewindow<Double>

SNMP packets that were dropped because they appeared outside of the authoritative SNMP engine's window.

snmpunknownusername<Double>

SNMP packets that were dropped because they referenced a user that was not known to the SNMP engine.

snmpunknownengineids<Double>

SNMP packets that were dropped because they referenced an SNMP engine ID that was not known to the NetScaler.

snmpwrongdigests<Double>

SNMP packets that were dropped because they did not contain the expected digest value.

snmpdecryptionerrors<Double>

SNMP packets that were dropped because they could not be decrypted.

Operations

You can perform the following operations:

get (all)

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Fetch statistics of the bound resources: NITRO allows you to fetch statistics of bound resources by specifying the "statbindings" query parameter as "yes". This query parameter is applicable only to get(byname) operation. For example, to get binding statistics of a given lbvserver, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/stat/lbvserver/lbvip1?statbindings=yes

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>

NOTE: Mandatory parameters are marked red and bold. Placeholder text is marked in green.

URL:http://<netscaler-ip-address>/nitro/v1/stat/snmp

Query-parameters:

args
