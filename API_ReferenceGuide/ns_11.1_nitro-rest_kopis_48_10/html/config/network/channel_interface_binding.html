channel_interface_binding

Binding object showing the interface that can be bound to channel.

Read/write properties

id<String>

ID of the LA channel or the cluster LA channel to which you want to bind interfaces. Specify an LA channel in LA/x notation, where x can range from 1 to 8 or a cluster LA channel in CLA/x notation or Link redundant channel in LR/x notation , where x can range from 1 to 4.

ifnum<String[]>

Interfaces to be bound to the LA channel of a NetScaler appliance or to the LA channel of a cluster configuration. For an LA channel of a NetScaler appliance, specify an interface in C/U notation (for example, 1/3). For an LA channel of a cluster configuration, specify an interface in N/C/U notation (for example, 2/1/3). where C can take one of the following values: * 0 - Indicates a management interface. * 1 - Indicates a 1 Gbps port. * 10 - Indicates a 10 Gbps port. U is a unique integer for representing an interface in a particular port group. N is the ID of the node to which an interface belongs in a cluster configuration. Use spaces to separate multiple entries.

Read only properties

lamode<String>

The mode(AUTO/MANNUAL) for the LA channel.
Possible values = MANUAL, AUTO

slaveduplex<Double>

Duplex of the member interfaces.

slaveflowctl<Double>

Flowcontrol of the member interfaces.

slavetime<Double>

UP time of the member interfaces.

slavestate<Double>

State of the member interfaces.

slavespeed<Double>

Speed of the member interfaces.

slavemedia<Double>

Media type of the member interfaces.

lractiveintf<Boolean>

LR set member interface state(active/inactive).

__count<Double>

count parameter

Operations

You can perform the following operations:

adddeletegetget (all)count

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
