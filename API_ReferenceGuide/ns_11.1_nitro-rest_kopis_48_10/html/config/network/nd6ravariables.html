nd6ravariables

Configuration for nd6 Router Advertisment configuration variables resource.

Read/write properties

vlan<Double>

The VLAN number.
Minimum value = 0
Maximum value = 4094

ceaserouteradv<String>

Cease router advertisements on this vlan.
Default value: NO
Possible values = YES, NO

sendrouteradv<String>

whether the router sends periodic RAs and responds to Router Solicitations.
Default value: NO
Possible values = YES, NO

srclinklayeraddroption<String>

Include source link layer address option in RA messages.
Default value: YES
Possible values = YES, NO

onlyunicastrtadvresponse<String>

Send only Unicast Router Advertisements in respond to Router Solicitations.
Default value: NO
Possible values = YES, NO

managedaddrconfig<String>

Value to be placed in the Managed address configuration flag field.
Default value: NO
Possible values = YES, NO

otheraddrconfig<String>

Value to be placed in the Other configuration flag field.
Default value: NO
Possible values = YES, NO

currhoplimit<Double>

Current Hop limit.
Default value: 64
Minimum value = 0
Maximum value = 255

maxrtadvinterval<Double>

Maximum time allowed between unsolicited multicast RAs, in seconds.
Default value: 600
Minimum value = 4
Maximum value = 1800

minrtadvinterval<Double>

Minimum time interval between RA messages, in seconds.
Default value: 198
Minimum value = 3
Maximum value = 1350

linkmtu<Double>

The Link MTU.
Default value: 0
Minimum value = 0
Maximum value = 1500

reachabletime<Double>

Reachable time, in milliseconds.
Default value: 0
Minimum value = 0
Maximum value = 3600000

retranstime<Double>

Retransmission time, in milliseconds.
Default value: 0

defaultlifetime<Integer>

Default life time, in seconds.
Default value: 1800
Minimum value = 0
Maximum value = 9000

Read only properties

lastrtadvtime<Double>

Last RA sent timestamp.

nextrtadvdelay<Double>

Next RA delay.

__count<Double>

count parameter

Operations

You can perform the following operations:

updateunsetget (all)getcount

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
