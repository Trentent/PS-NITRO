l3param

Configuration for Layer 3 related parameter resource.

Read/write properties

srcnat<String>

Perform NAT if only the source is in the private network.
Default value: ENABLED
Possible values = ENABLED, DISABLED

icmpgenratethreshold<Double>

NS generated ICMP pkts per 10ms rate threshold.
Default value: 100

overridernat<String>

USNIP/USIP settings override RNAT settings for configured
service/virtual server traffic.. .
Default value: DISABLED
Possible values = ENABLED, DISABLED

dropdfflag<String>

Enable dropping the IP DF flag.
Default value: DISABLED
Possible values = ENABLED, DISABLED

miproundrobin<String>

Enable round robin usage of mapped IPs.
Default value: ENABLED
Possible values = ENABLED, DISABLED

externalloopback<String>

Enable external loopback.
Default value: DISABLED
Possible values = ENABLED, DISABLED

tnlpmtuwoconn<String>

Enable/Disable learning PMTU of IP tunnel when ICMP error does not contain connection information.
Default value: ENABLED
Possible values = ENABLED, DISABLED

usipserverstraypkt<String>

Enable detection of stray server side pkts in USIP mode.
Default value: DISABLED
Possible values = ENABLED, DISABLED

forwardicmpfragments<String>

Enable forwarding of ICMP fragments.
Default value: DISABLED
Possible values = ENABLED, DISABLED

dropipfragments<String>

Enable dropping of IP fragments.
Default value: DISABLED
Possible values = ENABLED, DISABLED

acllogtime<Double>

Parameter to tune acl logging time.
Default value: 5000

icmperrgenerate<String>

Enable/Disable fragmentation required icmp error generation, before encapsulating a packet with vPath header. This knob is only functional for vPath Environment.
Default value: ENABLED
Possible values = ENABLED, DISABLED

implicitaclallow<String>

Do not apply ACLs for internal ports.
Default value: ENABLED
Possible values = ENABLED, DISABLED

dynamicrouting<String>

Enable/Disable Dynamic routing on partition.
Default value: DISABLED
Possible values = ENABLED, DISABLED

ipv6dynamicrouting<String>

Enable/Disable IPv6 Dynamic routing on partition.
Default value: DISABLED
Possible values = ENABLED, DISABLED

Operations

You can perform the following operations:

updateunsetget (all)

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
