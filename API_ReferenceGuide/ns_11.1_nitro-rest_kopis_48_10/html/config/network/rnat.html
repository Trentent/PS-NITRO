rnat

Configuration for RNAT configured route resource.

Read/write properties

network<String>

The network address defined for the RNAT entry.
Minimum length = 1

netmask<String>

The subnet mask for the network address.
Minimum length = 1

aclname<String>

An extended ACL defined for the RNAT entry.
Minimum length = 1

redirectport<Boolean>

The port number to which the packets are redirected.

natip<String>

The NAT IP address defined for the RNAT entry. .
Minimum length = 1

td<Double>

Integer value that uniquely identifies the traffic domain in which you want to configure the entity. If you do not specify an ID, the entity becomes part of the default traffic domain, which has an ID of 0.
Minimum value = 0
Maximum value = 4094

ownergroup<String>

The owner node group in a Cluster for this rnat rule.
Default value: DEFAULT_NG
Minimum length = 1

natip2<String>

The NAT IP(s) assigned to the RNAT.
Minimum length = 1

srcippersistency<String>

Enables the NetScaler appliance to use the same NAT IP address for all RNAT sessions initiated from a particular server.
Default value: DISABLED
Possible values = ENABLED, DISABLED

useproxyport<String>

Enable source port proxying, which enables the NetScaler appliance to use the RNAT ips using proxied source port.
Default value: ENABLED
Possible values = ENABLED, DISABLED

connfailover<String>

Synchronize connection information with the secondary appliance in a high availability (HA) pair. That is, synchronize all connection-related information for the RNAT session. In order for this to work, tcpproxy should be DISABLED. To disable tcpproxy use "set rnatparam tcpproxy DISABLED".
Default value: DISABLED
Possible values = ENABLED, DISABLED

Read only properties

__count<Double>

count parameter

Operations

You can perform the following operations:

clearupdateunsetget (all)count

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>

Use this query-parameter to get the filtered set of rnat resources configured on NetScaler.Filtering can be done on any of the properties of the resource.
