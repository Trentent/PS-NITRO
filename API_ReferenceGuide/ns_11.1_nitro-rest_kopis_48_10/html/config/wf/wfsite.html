wfsite

Configuration for WF site resource.

Read/write properties

sitename<String>

Name of the WebFront site being created on the NetScaler appliance.
Minimum length = 1
Maximum length = 255

storefronturl<String>

FQDN or IP of Windows StoreFront server where the store is configured.
Minimum length = 1
Maximum length = 255

storename<String>

Name of the store present in StoreFront.
Minimum length = 1
Maximum length = 255

html5receiver<String>

Specifies whether or not to use HTML5 receiver for launching apps for the WF site.
Default value: FALLBACK
Possible values = ALWAYS, FALLBACK, OFF

workspacecontrol<String>

Specifies whether of not to use workspace control for the WF site.
Default value: ON
Possible values = ON, OFF

displayroamingaccounts<String>

Specifies whether or not to display the accounts selection screen during First Time Use of Receiver .
Default value: ON
Possible values = ON, OFF

xframeoptions<String>

The value to be sent in the X-Frame-Options header. WARNING: Setting this option to ALLOW could leave the end user vulnerable to Click Jacking attacks.
Default value: DENY
Possible values = ALLOW, DENY

Read only properties

state<String>

State of wf site.
Default value: UP
Possible values = UP, INITIALIZING, DOWN-HostUnknown, DOWN-ReqTimeout, DOWN-Wrong Store, DOWN-SF Error, DOWN-SSL Error, DOWN-Conn Reset, DOWN

__count<Double>

count parameter

Operations

You can perform the following operations:

adddeleteupdateget (all)getcount

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
