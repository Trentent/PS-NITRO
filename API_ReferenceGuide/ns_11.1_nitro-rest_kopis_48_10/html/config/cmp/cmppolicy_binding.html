cmppolicy_binding

Binding object which returns the resources bound to cmppolicy.

Read/write properties

name<String>

Name of the HTTP compression policy for which to display details.
Minimum length = 1

Read only properties

cmppolicy_lbvserver_binding<cmppolicy_lbvserver_binding[]>

lbvserver that can be bound to cmppolicy.

cmppolicy_cmpglobal_binding<cmppolicy_cmpglobal_binding[]>

cmpglobal that can be bound to cmppolicy.

cmppolicy_crvserver_binding<cmppolicy_crvserver_binding[]>

crvserver that can be bound to cmppolicy.

cmppolicy_csvserver_binding<cmppolicy_csvserver_binding[]>

csvserver that can be bound to cmppolicy.

cmppolicy_cmppolicylabel_binding<cmppolicy_cmppolicylabel_binding[]>

cmppolicylabel that can be bound to cmppolicy.

Operations

You can perform the following operations:

getget (all)

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>

get (all)

URL:http://<netscaler-ip-address>/nitro/v1/config/cmppolicy_binding

HTTP Method:GET

Request Headers:

Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
Accept:application/json

Query-parameters:

bulkbindings

http://<netscaler-ip-address>/nitro/v1/config/cmppolicy_binding?bulkbindings=yes

NITRO allows you to fetch bindings in bulk.
Note: get (all) method can be used only in conjuction with bulkbindings query parameter.
