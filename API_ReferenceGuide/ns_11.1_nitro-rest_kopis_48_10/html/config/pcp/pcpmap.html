pcpmap

Configuration for server resource.

Read only properties

pcpsrcip<String>

Internal IP address.

pcpsrcport<Integer>

Internal Port.
Range 1 - 65535
* in CLI is represented as 65535 in NITRO API

pcpdstip<String>

Remote PEER IP address.

pcpdstport<Integer>

Remote PEER Port.
Range 1 - 65535
* in CLI is represented as 65535 in NITRO API

pcpnatip<String>

External IP address.

pcpnatport<Integer>

External Port Number.
Range 1 - 65535
* in CLI is represented as 65535 in NITRO API

pcpprotocol<Double>

Protocol of the PCP mapping.

pcpaddr<Double>

Address of PCP Map entity.

pcprefcnt<Double>

Refcount, which indicates how many session is active.

__count<Double>

count parameter

Operations

You can perform the following operations:

get (all)count

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>

get (all)

URL:http://<netscaler-ip-address>/nitro/v1/config/pcpmap

Query-parameters:

attrs

http://<netscaler-ip-address>/nitro/v1/config/pcpmap?attrs=property-name1,property-name2

Use this query parameter to specify the resource details that you want to retrieve.

filter

http://<netscaler-ip-address>/nitro/v1/config/pcpmap?filter=property-name1:property-val1,property-name2:property-val2

Use this query-parameter to get the filtered set of pcpmap resources configured on NetScaler.Filtering can be done on any of the properties of the resource.

view

http://<netscaler-ip-address>/nitro/v1/config/pcpmap?view=summary

Use this query-parameter to get the summary output of pcpmap resources configured on NetScaler.

Note: By default, the retrieved results are displayed in detail view (?view=detail).

pagination

http://<netscaler-ip-address>/nitro/v1/config/pcpmap?pagesize=#no&pageno=#no

Use this query-parameter to get the pcpmap resources in chunks.
