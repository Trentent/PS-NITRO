lsnclient_binding

Binding object which returns the resources bound to lsnclient.

Read/write properties

clientname<String>

Name for the LSN client entity. Must begin with an ASCII alphanumeric or underscore (_) character, and must contain only ASCII alphanumeric, underscore, hash (#), period (.), space, colon (:), at (@), equals (=), and hyphen (-) characters. Cannot be changed after the LSN client is created. The following requirement applies only to the NetScaler CLI: If the name includes one or more spaces, enclose the name in double or single quotation marks (for example, "lsn client1" or 'lsn client1'). .
Minimum length = 1
Maximum length = 127

Read only properties

lsnclient_network6_binding<lsnclient_network6_binding[]>

network6 that can be bound to lsnclient.

lsnclient_nsacl6_binding<lsnclient_nsacl6_binding[]>

nsacl6 that can be bound to lsnclient.

lsnclient_network_binding<lsnclient_network_binding[]>

network that can be bound to lsnclient.

lsnclient_nsacl_binding<lsnclient_nsacl_binding[]>

nsacl that can be bound to lsnclient.

Operations

You can perform the following operations:

getget (all)

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
