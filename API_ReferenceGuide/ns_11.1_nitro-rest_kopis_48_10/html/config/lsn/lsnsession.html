lsnsession

Configuration for lsn session resource.

Read/write properties

nattype<String>

Type of sessions to be displayed.
Default value: NAT44
Possible values = NAT44, DS-Lite, NAT64

clientname<String>

Name of the LSN Client entity.

network<String>

IP address or network address of subscriber(s).

netmask<String>

Subnet mask for the IP address specified by the network parameter.

network6<String>

IPv6 address of the LSN subscriber or B4 device.

td<Double>

Traffic domain ID of the LSN client entity.
Default value: 0
Minimum value = 0
Maximum value = 4094

natip<String>

Mapped NAT IP address used in LSN sessions.

natport2<Integer>

Mapped NAT port used in the LSN sessions.

Read only properties

natprefix<String>

IPv6 address of the LSN subscriber(s) or subscriber network(B4-Device) on whose traffic the NetScaler ADC perform Large Scale NAT.

subscrip<String>

The Source IP address.

subscrport<Integer>

The Source Port.

destip<String>

The Destination IP address.

destport<Integer>

The Destination Port.

natport<Integer>

The NAT Port.

transportprotocol<String>

The Transport Protocol for the session.
Possible values = TCP, UDP, ICMP

sessionestdir<String>

The Session establishment direction, session was established for outbound or inbound packet.
Possible values = OUT, IN

dsttd<Double>

.

srctd<Double>

.

ipv6address<String>

IPv6 address of v6 vserver.

__count<Double>

count parameter

Operations

You can perform the following operations:

get (all)countflush

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
