vpnformssoaction

Configuration for Form sso action resource.

Read/write properties

name<String>

Name for the form based single sign-on profile.
Minimum length = 1

actionurl<String>

Root-relative URL to which the completed form is submitted.
Minimum length = 1

userfield<String>

Name of the form field in which the user types in the user ID.
Minimum length = 1

passwdfield<String>

Name of the form field in which the user types in the password.
Minimum length = 1

ssosuccessrule<String>

Advanced expression that defines the criteria for SSO success. Expression such as checking for cookie in the response is a common example.

namevaluepair<String>

Other name-value pair attributes to send to the server, in addition to sending the user name and password. Value names are separated by an ampersand (&), such as in name1=value1&name2=value2.

responsesize<Double>

Maximum number of bytes to allow in the response size. Specifies the number of bytes in the response to be parsed for extracting the forms.
Default value: 8096

nvtype<String>

How to process the name-value pair. Available settings function as follows:
* STATIC - The administrator-configured values are used.
* DYNAMIC - The response is parsed, the form is extracted, and then submitted.
Default value: DYNAMIC
Possible values = STATIC, DYNAMIC

submitmethod<String>

HTTP method (GET or POST) used by the single sign-on form to send the logon credentials to the logon server.
Default value: GET
Possible values = GET, POST

Read only properties

__count<Double>

count parameter

Operations

You can perform the following operations:

adddeleteupdateunsetget (all)getcount

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
