callhome

Configuration for callhome resource.

Read/write properties

emailaddress<String>

Email address of the contact administrator.
Maximum length = 78

proxymode<String>

Enables or disables the proxy mode. The proxy server can be set by either specifying the IP address of the server or the name of the service representing the proxy server.
Default value: NO
Possible values = YES, NO

ipaddress<String>

IP address of the proxy server.
Minimum length = 1

proxyauthservice<String>

Name of the service that represents the proxy server.
Maximum length = 128

port<Integer>

HTTP port on the Proxy server. This is a mandatory parameter for both IP address and service name based configuration.
Minimum value = 1
Range 1 - 65535
* in CLI is represented as 65535 in NITRO API

Read only properties

sslcardfirstfailure<String>

First occurrence SSL card failure.

sslcardlatestfailure<String>

Latest occurrence SSL card failure.

powfirstfail<String>

First occurrence power supply unit failure.

powlatestfailure<String>

Latest occurrence power supply unit failure.

hddfirstfail<String>

First occurrence hard disk drive failure.

hddlatestfailure<String>

Latest occurrence hard disk drive failure.

flashfirstfail<String>

First occurrence compact flash failure.

flashlatestfailure<String>

Latest occurrence compact flush failure.

restartlatestfail<String>

Latest occurrence warm restart failure.

callhomestatus<String[]>

Callhome feature enabled/disable, register with upload server successful/failed.
Possible values = ENABLED, DISABLED, SUCCESSFUL, FAILED

Operations

You can perform the following operations:

get (all)updateunset

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
