clusternode

Configuration for cluster node resource.

Read/write properties

nodeid<Double>

Unique number that identifies the cluster node.
Minimum value = 0
Maximum value = 31

ipaddress<String>

NetScaler IP (NSIP) address of the appliance to add to the cluster. Must be an IPv4 address.
Minimum length = 1

state<String>

Admin state of the cluster node. The available settings function as follows:
ACTIVE - The node serves traffic.
SPARE - The node does not serve traffic unless an ACTIVE node goes down.
PASSIVE - The node does not serve traffic, unless you change its state. PASSIVE state is useful during temporary maintenance activities in which you want the node to take part in the consensus protocol but not to serve traffic.
Default value: PASSIVE
Possible values = ACTIVE, SPARE, PASSIVE

backplane<String>

Interface through which the node communicates with the other nodes in the cluster. Must be specified in the three-tuple form n/c/u, where n represents the node ID and c/u refers to the interface on the appliance.
Minimum length = 1

priority<Double>

Preference for selecting a node as the configuration coordinator. The node with the lowest priority value is selected as the configuration coordinator.
When the current configuration coordinator goes down, the node with the next lowest priority is made the new configuration coordinator. When the original node comes back up, it will preempt the new configuration coordinator and take over as the configuration coordinator.
Note: When priority is not configured for any of the nodes or if multiple nodes have the same priority, the cluster elects one of the nodes as the configuration coordinator.
Default value: 31
Minimum value = 0
Maximum value = 31

nodegroup<String>

The default node group in a Cluster system.
Default value: DEFAULT_NG
Minimum length = 1

clearnodegroupconfig<String>

Option to remove nodegroup config.
Default value: YES
Possible values = YES, NO

Read only properties

clusterhealth<String>

Node clusterd state.
Possible values = UP, Configurations of the node are lagging behind the cluster by more than 256 commands, The config sync operation has exceeded the time limit of 60 seconds, The config sync operation (full sync) is in progress, The node is not in sync with the cluster configurations as sync is disabled on this node, The execution of a configuration command has failed on this node, UNKNOWN

effectivestate<String>

Node effective health state.
Possible values = UP, NOT UP, UNKNOWN, INIT

operationalsyncstate<String>

Node Operational Reconciliation state.
Default value: ENABLED
Possible values = ENABLED, SUCCESS, IN PROGRESS, FAILED, INCREMENTAL SYNC DISABLED, DISABLED, UNKNOWN

masterstate<String>

Node Master state.
Possible values = INACTIVE, ACTIVE, UNKNOWN

health<String>

Node Health state.
Possible values = UNKNOWN, INIT, DOWN, UP, Some enabled and HAMON interfaces of the node are down, All interfaces of the node are down or disabled, SSL card(s) of the node have failed, Route Monitor(s) of the node have failed, Service state is being synchronized with the cluster, The backplane interface is not set, is down, or is disabled, The CLAG member(s) of the node are down, Persistence sessions are being synchronized with the cluster, The Syn Cookie is being synchronized with the cluster, Unknown Health, AAA keys are being sychronized with the cluster

isconfigurationcoordinator<Boolean>

This argument is used to determine whether the node is configuration coordinator (CCO).

islocalnode<Boolean>

This argument is used to determine whether it is local node.

nodersskeymismatch<Boolean>

This argument is used to determine if there is a RSS key mismatch at cluster node level.

nodelicensemismatch<Boolean>

This argument is used to determine if there is a License mismatch at cluster node level.

nodejumbonotsupported<Boolean>

This argument is used to determine if Jumbo framework not supported at cluster node level.

nodelist<Double[]>

Nodelist for displaying Heartbeat not seen interfaces on a cluster node.

ifaceslist<String[]>

Interface list corresponding to nodelist for Heartbeat not seen interfaces on a cluster node.

enabledifaces<String>

Enabled Interfaces on a cluster node.

disabledifaces<String>

Disabled Interfaces on a cluster node.

partialfailifaces<String>

Partial Failure Interfaces on a cluster node.

hamonifaces<String>

Hamon Interfaces on a cluster node.

name<String>

Name of the state specific nodegroup.
Minimum length = 1

cfgflags<Double>

Flag indicates whether the node is bound to cluster nodegroup.

routemonitor<String>

The IP address (IPv4 or IPv6).

netmask<String>

The netmask.

__count<Double>

count parameter

Operations

You can perform the following operations:

addupdateunsetdeleteget (all)getcount

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
