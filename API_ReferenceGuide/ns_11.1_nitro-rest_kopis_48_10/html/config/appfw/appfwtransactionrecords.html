appfwtransactionrecords

Configuration for Application firewall transaction record resource.

Read only properties

httptransactionid<Double>

The http transaction identifier.

packetengineid<Integer>

The packet engine identifier.

appfwsessionid<String>

The session identifier set by the Application Firewall to track the user session.

profilename<String>

Application Firewall profile name.

url<String>

Request URL.

clientip<String>

The IP address of client.

destip<String>

The IP address of destination.

starttime<String>

Conveys time at which request processing started.

endtime<String>

Conveys time at which request processing end.

requestcontentlength<Integer>

The content length of request.

requestyields<Double>

The number of times yielded during request processing to send heart beat packets.

requestmaxprocessingtime<Double>

The maximum processing time across yields during request processing.

responsecontentlength<Integer>

The content length of response.

responseyields<Double>

The number of times yielded during response processing to send heart beat packets.

responsemaxprocessingtime<Double>

The maximum processing time across yields during response processing.

__count<Double>

count parameter

Operations

You can perform the following operations:

get (all)count

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
