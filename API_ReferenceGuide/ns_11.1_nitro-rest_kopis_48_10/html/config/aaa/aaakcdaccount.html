aaakcdaccount

Configuration for Kerberos constrained delegation account resource.

Read/write properties

kcdaccount<String>

The name of the KCD account.
Minimum length = 1

keytab<String>

The path to the keytab file. If specified other parameters in this command need not be given.

realmstr<String>

Kerberos Realm.

delegateduser<String>

Username that can perform kerberos constrained delegation.

kcdpassword<String>

Password for Delegated User.

usercert<String>

SSL Cert (including private key) for Delegated User.

cacert<String>

CA Cert for UserCert or when doing PKINIT backchannel.

userrealm<String>

Realm of the user.

enterpriserealm<String>

Enterprise Realm of the user. This should be given only in certain KDC deployments where KDC expects Enterprise username instead of Principal Name.

servicespn<String>

Service SPN. When specified, this will be used to fetch kerberos tickets. If not specified, Netscaler will construct SPN using service fqdn.

Read only properties

principle<String>

SPN extracted from keytab file.

kcdspn<String>

Host SPN extracted from keytab file.

__count<Double>

count parameter

Operations

You can perform the following operations:

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
