ipsecparameter

Configuration for IPSEC paramter resource.

Read/write properties

ikeversion<String>

IKE Protocol Version.
Default value: V2
Possible values = V1, V2

encalgo<String[]>

Type of encryption algorithm.
Default value: AES
Possible values = AES, 3DES

hashalgo<String[]>

Type of hashing algorithm.
Default value: HMAC_SHA256
Possible values = HMAC_SHA1, HMAC_SHA256, HMAC_SHA384, HMAC_SHA512, HMAC_MD5

lifetime<Double>

Lifetime of IKE SA in seconds. Lifetime of IPSec SA will be (lifetime of IKE SA/8).
Minimum value = 480
Maximum value = 31536000

livenesscheckinterval<Double>

Number of seconds after which a notify payload is sent to check the liveliness of the peer. Additional retries are done as per retransmit interval setting. Zero value disables liveliness checks.
Minimum value = 0
Maximum value = 64999

replaywindowsize<Double>

IPSec Replay window size for the data traffic.
Minimum value = 0
Maximum value = 16384

ikeretryinterval<Double>

IKE retry interval for bringing up the connection.
Minimum value = 60
Maximum value = 3600

perfectforwardsecrecy<String>

Enable/Disable PFS.
Default value: DISABLE
Possible values = ENABLE, DISABLE

retransmissiontime<Double>

The interval in seconds to retry sending the IKE messages to peer, three consecutive attempts are done with doubled interval after every failure,
increases for every retransmit till 6 retransmits.
Minimum value = 1
Maximum value = 99

Read only properties

responderonly<String>

Responder Only config for IKED.
Default value: NO
Possible values = YES, NO

Operations

You can perform the following operations:

updateunsetget (all)

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
