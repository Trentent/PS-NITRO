gslbdomain_gslbvserver_binding

Binding object showing the gslbvserver that can be bound to gslbdomain.

Read/write properties

name<String>

Name of the Domain.
Minimum length = 1

vservername<String>

.

Read only properties

sitename<String>

Name of the site to which the service belongs.

backuplbmethod<String>

Indicates the backup method in case the primary fails.
Possible values = ROUNDROBIN, LEASTCONNECTION, LEASTRESPONSETIME, SOURCEIPHASH, LEASTBANDWIDTH, LEASTPACKETS, STATICPROXIMITY, RTT, CUSTOMLOAD

customheaders<String>

The string that is sent to the service. Applicable to HTTP ,HTTP-ECV and RTSP monitor types.

persistenceid<Double>

Persistence id of the gslb vserver.

state<String>

The state of the vserver.
Possible values = UP, DOWN, UNKNOWN, BUSY, OUT OF SERVICE, GOING OUT OF SERVICE, DOWN WHEN GOING OUT OF SERVICE, NS_EMPTY_STR, Unknown, DISABLED

lbmethod<String>

The load balancing method set for the virtual server.
Possible values = ROUNDROBIN, LEASTCONNECTION, LEASTRESPONSETIME, SOURCEIPHASH, LEASTBANDWIDTH, LEASTPACKETS, STATICPROXIMITY, RTT, CUSTOMLOAD

edr<String>

Send clients an empty DNS response when the GSLB virtual server is DOWN.
Default value: DISABLED
Possible values = ENABLED, DISABLED

cip<String>

Indicates if Client IP option is enabled.
Possible values = ENABLED, DISABLED

statechangetimesec<String>

Time since last state change.

netmask<String>

Netmask.
Minimum length = 1

dnsrecordtype<String>

The IP type for this GSLB vserver.
Possible values = A, AAAA, CNAME, NAPTR

persistmask<String>

The optional IPv4 network mask applied to IPv4 addresses to establish source IP address based persistence.
Minimum length = 1

sitepersistence<String>

Indicates the type of cookie persistence set.
Possible values = ConnectionProxy, HTTPRedirect, NONE

servicetype<String>

The type GSLB service.
Possible values = HTTP, FTP, TCP, UDP, SSL, SSL_BRIDGE, SSL_TCP, NNTP, ANY, SIP_UDP, SIP_TCP, SIP_SSL, RADIUS, RDP, RTSP, MYSQL, MSSQL, ORACLE

v6persistmasklen<Double>

Number of bits to consider in an IPv6 source IP address when creating source IP address based persistence sessions.
Default value: 128
Minimum value = 1
Maximum value = 128

v6netmasklen<Double>

Number of bits to consider, in an IPv6 source IP address, for creating the hash that is required by the SOURCEIPHASH load balancing method.
Default value: 128
Minimum value = 1
Maximum value = 128

persistencetype<String>

Indicates if persistence is set on the gslb vserver.
Possible values = SOURCEIP, NONE

siteprefix<String>

The site prefix string.

dynamicweight<String>

Dynamic weight method of the vserver.
Default value: DISABLED
Possible values = SERVICECOUNT, SERVICEWEIGHT, DISABLED

mir<String>

.
Default value: DISABLED
Possible values = ENABLED, DISABLED

__count<Double>

count parameter

Operations

You can perform the following operations:

getget (all)count

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
