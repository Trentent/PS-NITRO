gslbdomain_lbmonitor_binding

Binding object showing the lbmonitor that can be bound to gslbdomain.

Read/write properties

name<String>

Name of the Domain.
Minimum length = 1

monitorname<String>

Monitor name.

Read only properties

monstatcode<Integer>

The code indicating the monitor response.

customheaders<String>

The string that is sent to the service. Applicable to HTTP ,HTTP-ECV and RTSP monitor types.

iptunnel<String>

The state of the monitor for tunneled devices.
Possible values = YES, NO

responsetime<Double>

Response time of this monitor.

monstate<String>

Monitor state.
Possible values = UP, DOWN, UNKNOWN, BUSY, OUT OF SERVICE, GOING OUT OF SERVICE, DOWN WHEN GOING OUT OF SERVICE, NS_EMPTY_STR, Unknown, DISABLED

lastresponse<String>

The string form of monstatcode.

servicename<String>

The service name.

monitortotalprobes<Double>

Total monitor probes.

respcode<String>

The response codes.

vservername<String>

.

httprequest<String>

HTTP request to the backend server.

monitortotalfailedprobes<Double>

Total probes failed.

monitorcurrentfailedprobes<Double>

Total number of current failed probes.

__count<Double>

count parameter

Operations

You can perform the following operations:

getget (all)count

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
