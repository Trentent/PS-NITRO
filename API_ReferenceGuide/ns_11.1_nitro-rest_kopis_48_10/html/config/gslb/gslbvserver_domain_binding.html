gslbvserver_domain_binding

Binding object showing the domain that can be bound to gslbvserver.

Read/write properties

backupipflag<Boolean>

The IP address of the backup service for the specified domain name. Used when all the services bound to the domain are down, or when the backup chain of virtual servers is down.

cookietimeout<Double>

Timeout, in minutes, for the GSLB site cookie.
Minimum value = 0
Maximum value = 1440

backupip<String>

The IP address of the backup service for the specified domain name. Used when all the services bound to the domain are down, or when the backup chain of virtual servers is down.
Minimum length = 1

name<String>

Name of the virtual server on which to perform the binding operation.
Minimum length = 1

ttl<Double>

Time to live (TTL) for the domain.
Minimum value = 1

domainname<String>

Domain name for which to change the time to live (TTL) and/or backup service IP address.
Minimum length = 1

sitedomainttl<Double>

TTL, in seconds, for all internally created site domains (created when a site prefix is configured on a GSLB service) that are associated with this virtual server.
Minimum value = 1

cookie_domainflag<Boolean>

The cookie domain for the GSLB site. Used when inserting the GSLB site cookie in the HTTP response.

cookie_domain<String>

The cookie domain for the GSLB site. Used when inserting the GSLB site cookie in the HTTP response.
Minimum length = 1

__count<Double>

count parameter

Operations

You can perform the following operations:

adddeletegetget (all)count

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
