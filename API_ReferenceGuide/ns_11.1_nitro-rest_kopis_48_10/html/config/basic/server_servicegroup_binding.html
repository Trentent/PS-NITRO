server_servicegroup_binding

Binding object showing the servicegroup that can be bound to server.

Read/write properties

name<String>

Name of the server for which to display parameters.
Minimum length = 1

servicegroupname<String>

servicegroups bind to this server.

Read only properties

sp<String>

Enable surge protection for the service group.
Default value: OFF
Possible values = ON, OFF

sc<String>

State of the SureConnect feature for the service group.
Default value: OFF
Possible values = ON, OFF

svrtimeout<Double>

Time, in seconds, after which to terminate an idle server connection.
Minimum value = 0
Maximum value = 31536000

serviceipstr<String>

This field has been intorduced to show the dbs services ip.

tcpb<String>

Enable TCP buffering for the service group.
Possible values = YES, NO

cip<String>

Before forwarding a request to the service, insert an HTTP header with the client's IPv4 or IPv6 address as its value. Used if the server needs the client's IP address for security, accounting, or other purposes, and setting the Use Source IP parameter is not a viable option.
Possible values = ENABLED, DISABLED

dup_port<Integer>

port of the service.
Range 1 - 65535
* in CLI is represented as 65535 in NITRO API

customserverid<String>

A positive integer to identify the service. Used when the persistency type is set to Custom Server ID.
Default value: "None"

boundtd<Double>

Integer value that uniquely identifies the traffic domain in which you want to configure the entity. If you do not specify an ID, the entity becomes part of the default traffic domain, which has an ID of 0.
Minimum value = 0
Maximum value = 4094

downstateflush<String>

Perform delayed clean-up of connections to all services in the service group.
Default value: ENABLED
Possible values = ENABLED, DISABLED

svctype<String>

The type of bound service.
Possible values = HTTP, FTP, TCP, UDP, SSL, SSL_BRIDGE, SSL_TCP, DTLS, NNTP, RPCSVR, DNS, ADNS, SNMP, RTSP, DHCPRA, ANY, SIP_UDP, SIP_TCP, SIP_SSL, DNS_TCP, ADNS_TCP, MYSQL, MSSQL, ORACLE, RADIUS, RADIUSListener, RDP, DIAMETER, SSL_DIAMETER, TFTP, SMPP, PPTP, GRE, SYSLOGTCP, SYSLOGUDP, FIX, SSL_FIX

cka<String>

Enable client keep-alive for the service group.
Possible values = YES, NO

svrstate<String>

The state of the bound service.
Possible values = UP, DOWN, UNKNOWN, BUSY, OUT OF SERVICE, GOING OUT OF SERVICE, DOWN WHEN GOING OUT OF SERVICE, NS_EMPTY_STR, Unknown, DISABLED

appflowlog<String>

Enable logging of AppFlow information for the specified service group.
Default value: ENABLED
Possible values = ENABLED, DISABLED

cipheader<String>

Name of the HTTP header whose value must be set to the IP address of the client. Used with the Client IP parameter. If client IP insertion is enabled, and the client IP header is not specified, the value of Client IP Header parameter or the value set by the set ns config command is used as client's IP header name.
Minimum length = 1

maxclient<Double>

Maximum number of simultaneous open connections for the service group.
Minimum value = 0
Maximum value = 4294967294

clttimeout<Double>

Time, in seconds, after which to terminate an idle client connection.
Minimum value = 0
Maximum value = 31536000

port<Integer>

The port number to be used for the bound service.
Range 1 - 65535
* in CLI is represented as 65535 in NITRO API

dup_svctype<String>

service type of the service.
Possible values = HTTP, FTP, TCP, UDP, SSL, SSL_BRIDGE, SSL_TCP, DTLS, NNTP, RPCSVR, DNS, ADNS, SNMP, RTSP, DHCPRA, ANY, SIP_UDP, SIP_TCP, SIP_SSL, DNS_TCP, ADNS_TCP, MYSQL, MSSQL, ORACLE, RADIUS, RADIUSListener, RDP, DIAMETER, SSL_DIAMETER, TFTP, SMPP, PPTP, GRE, SYSLOGTCP, SYSLOGUDP, FIX, SSL_FIX

cmp<String>

Enable compression for the specified service.
Possible values = YES, NO

monthreshold<Double>

Minimum sum of weights of the monitors that are bound to this service. Used to determine whether to mark a service as UP or DOWN.
Minimum value = 0
Maximum value = 65535

maxreq<Double>

Maximum number of requests that can be sent on a persistent connection to the service group. Note: Connection requests beyond this value are rejected.
Minimum value = 0
Maximum value = 65535

serviceipaddress<String>

The IP address of the bound service.

usip<String>

Use the client's IP address as the source IP address when initiating a connection to the server. When creating a service, if you do not set this parameter, the service inherits the global Use Source IP setting (available in the enable ns mode and disable ns mode CLI commands, or in the System > Settings > Configure modes > Configure Modes dialog box). However, you can override this setting after you create the service.
Possible values = YES, NO

cacheable<String>

Use the transparent cache redirection virtual server to forward the request to the cache server.
Default value: NO
Possible values = YES, NO

maxbandwidth<Double>

Maximum bandwidth, in Kbps, allocated for all the services in the service group.
Minimum value = 0
Maximum value = 4294967287

svrcfgflags<Double>

service flags to denote its a db enabled.

__count<Double>

count parameter

Operations

You can perform the following operations:

getget (all)count

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
