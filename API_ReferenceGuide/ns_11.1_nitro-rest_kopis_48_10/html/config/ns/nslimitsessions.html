nslimitsessions

Configuration for limit sessions resource.

Read/write properties

limitidentifier<String>

Name of the rate limit identifier for which to display the sessions.
Minimum length = 1

detail<Boolean>

Show the individual hash values.

Read only properties

timeout<Double>

The time remaining on the session before a flush can be attempted. If active transactions are present the session will not be flushed.

hits<Double>

The number of times this entry was hit.

drop<Double>

The number of times action was taken.

number<Double[]>

The hash of the matched selectlets.

name<String>

The string formed by gathering selectlet values.

unit<Double>

Total computed hash of the matched selectlets.

flags<Double>

Used internally to identify ip addresses.

referencecount<Double>

Total number of transactions pointing to this entry. Its the sum total of the connection and bandwidth references.

maxbandwidth<Double>

The current bandwidth.

selectoripv61<String>

First IPV6 address gathered.

selectoripv62<String>

Second IPV6 address gathered.

flag<Double>

Used internally to identify ipv6 addresses.

__count<Double>

count parameter

Operations

You can perform the following operations:

get (all)countclear

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
