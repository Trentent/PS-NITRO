nspartition

Configuration for admin partition resource.

Read/write properties

partitionname<String>

Name of the Partition. Must begin with an ASCII alphanumeric or underscore (_) character, and must contain only ASCII alphanumeric, underscore, hash (#), period (.), space, colon (:), at (@), equals (=), and hyphen (-) characters.
Minimum length = 1

maxbandwidth<Double>

Maximum bandwidth in Kbps, permitted on this partition.
Default value: 10240

minbandwidth<Double>

Minimum bandwidth in Kbps, permitted on this partition.
Default value: 10240

maxconn<Double>

Maximum number of connections permitted on this partition.
Default value: 1024

maxmemlimit<Double>

Maximum memory allowed for this partition in megabytes.
Default value: 10
Minimum value = 5

partitionmac<String>

Special MAC address for the partition which is used for communication over shared vlans in this partition. If not specified, the MAC address is auto-generated.

Read only properties

partitionid<Double>

Partition Id.
Minimum value = 1

partitiontype<String>

Type of the Partition.
Possible values = Default Partition, Current Partition

pmacinternal<Boolean>

Partition MAC is generated internally.

__count<Double>

count parameter

Operations

You can perform the following operations:

adddeleteupdateunsetSwitchget (all)getcount

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
