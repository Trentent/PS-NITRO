nshttpparam

Configuration for HTTP parameter resource.

Read/write properties

dropinvalreqs<String>

Drop invalid HTTP requests or responses.
Default value: OFF
Possible values = ON, OFF

markhttp09inval<String>

Mark HTTP/0.9 requests as invalid.
Default value: OFF
Possible values = ON, OFF

markconnreqinval<String>

Mark CONNECT requests as invalid.
Default value: OFF
Possible values = ON, OFF

insnssrvrhdr<String>

Enable or disable NetScaler server header insertion for NetScaler generated HTTP responses.
Default value: OFF
Possible values = ON, OFF

nssrvrhdr<String>

The server header value to be inserted. If no explicit header is specified then NSBUILD.RELEASE is used as default server header.
Minimum length = 1

logerrresp<String>

Server header value to be inserted.
Default value: ON
Possible values = ON, OFF

conmultiplex<String>

Reuse server connections for requests from more than one client connections.
Default value: ENABLED
Possible values = ENABLED, DISABLED

maxreusepool<Double>

Maximum limit on the number of connections, from the NetScaler to a particular server that are kept in the reuse pool. This setting is helpful for optimal memory utilization and for reducing the idle connections to the server just after the peak time.
Minimum value = 0
Maximum value = 360000

Read only properties

builtin<String[]>

Flag to determine if the http param is built-in or not.
Possible values = MODIFIABLE, DELETABLE, IMMUTABLE, PARTITION_ALL

Operations

You can perform the following operations:

updateunsetget (all)

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
