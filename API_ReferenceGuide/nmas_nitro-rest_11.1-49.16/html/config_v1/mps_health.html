mps_health

Configuration for Health Monitoring Stats resource.

Read/write properties

disks_usage<mps_disk_usage[]>

Individual Disks Usage.

node_id<String>

Application Node ID.

id<String>

Id is system generated key.

Read only properties

memory_total<Double>

Total memory in KB.

disk_total_capacity<Double>

Disk Total Capacity in KB.

disk_usage<Double>

Disk Usage (%).

disk_total<Double>

Total available disk space in KB.

memory_usage<Double>

Memory Usage (%).

disk_free<Double>

Free disk available in KB.

page_size<Double>

Total memory in KB.

memory_free<Double>

Free memory available in KB.

cpu_usage<Double>

CPU Usage (%).

disk_used<Double>

Disk used in KB.

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE

get (all)

URL:http://<MGMT-IP>/nitro/v1/config/mps_health

Query-parameters:

filter

http://<MGMT-IP>/nitro/v1/config/mps_health?filter=property-name1:property-value1,property-name2:property-value2

Use this query-parameter to get the filtered set of mps_health resources configured on the system. You can set a filter on any property of the resource.

pagesize=#no&pageno=#no

http://<MGMT-IP>/nitro/v1/config/mps_health?pagesize=#no&pageno=#no

Use this query-parameter to get the mps_health resources in chunks.

count

http://<MGMT-IP>/nitro/v1/config/mps_health?count=yes

Use this query-parameter to get the count of mps_health resources.
