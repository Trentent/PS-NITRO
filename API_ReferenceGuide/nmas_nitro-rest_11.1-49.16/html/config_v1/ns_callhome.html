ns_callhome

Configuration for NS Callhome Information resource.

Read/write properties

proxy_mode<String>

Deploy the callhome proxy mode.Possible values YES or NO.
Maximum length = 78

proxy_server_ip<String>

Proxy Server IP address.
Minimum length = 1
Maximum length = 64

ip_address<String>

NS IP Address for callhome.
Minimum length = 1
Maximum length = 64

proxy_server_port<Integer>

Proxy server Port.
Maximum value =

email_address<String>

The contact person's E-mail address.
Maximum length = 78

name<String>

Name of the vpx Instance.
Maximum length = 78

ns_ip_address_arr<String[]>

List of NS IP Address.
Minimum length = 1
Maximum length = 64

Read only properties

powfirstfail<String>

First occurrence power supply unit failure.

callhomesenablestatus<String>

Callhome feature enabled/disable.

powlatestfail<String>

Latest occurrence power supply unit failure.

flashlatestfail<String>

Latest occurrence compact flash failure.

restartfirstfail<String>

First occurrence warm restart failure.

hddlatestfail<String>

Latest occurrence hard disk drive failure.

flashfirstfail<String>

First occurrence compact flash failure.

restartlatestfail<String>

Latest occurrence warm restart failure.

hddfirstfail<String>

First occurrence hard disk drive failure.

callhomestatus<String>

Callhome feature register with upload server successful/failed.

sslcardfirstfailure<String>

First occurrence SSL card failure.

sslcardlatestfailure<String>

Latest occurrence SSL card failure.

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE
