mas_license

Configuration for MAS License Information resource.

Read/write properties

cpx_lic<Integer>

CPX Licenses, 0=Unlicensed/Disabled, 1=Unlicensed/Enabled, 2=Licensed/Disabled, 3=Licensed/Enabled.

analytics<Integer>

Analytics, 0=Unlicensed/Disabled, 1=Unlicensed/Enabled, 2=Licensed/Disabled, 3=Licensed/Enabled.

adv_analytics<Integer>

Advanced Analytics, 0=Unlicensed/Disabled, 1=Unlicensed/Enabled, 2=Licensed/Disabled, 3=Licensed/Enabled.

perf<Integer>

Perf, 0=Unlicensed/Disabled, 1=Unlicensed/Enabled, 2=Licensed/Disabled, 3=Licensed/Enabled.

syslog<Integer>

Syslog, 0=Unlicensed/Disabled, 1=Unlicensed/Enabled, 2=Licensed/Disabled, 3=Licensed/Enabled.

id<String>

Id is system generated key.

pooled_lic<Integer>

Pooled Licenses, 0=Unlicensed/Disabled, 1=Unlicensed/Enabled, 2=Licensed/Disabled, 3=Licensed/Enabled.

snmp_traps<Integer>

SNMP Traps, 0=Unlicensed/Disabled, 1=Unlicensed/Enabled, 2=Licensed/Disabled, 3=Licensed/Enabled.

Read only properties

max_vips<String>

Maximum VIPs.

node_id<String>

Node ID.

rpt_sampletime<Double>

Time Stamp.

total_managed_vips<Integer>

Total managed VIPs.

total_discovered_vips<Integer>

Total discovered VIPs.

total_allowed_vips<Integer>

Total Allowed VIPs.

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE
