mps

Configuration for System Status resource.

Read/write properties

is_cloud<Boolean>

True if its a cloud deployment.

is_member_of_default_group<Boolean>

Is Member Of Default Group.

privilege_feature<String>

privilege_feature.

Read only properties

build_number_short<String>

Build Number without Date.

current_time<Integer>

Current Time.

sysid<String>

System Id.

hostname<String>

Host name on which system is running.

bios_version<String>

BIOS Version.

hostid<String>

Host Id.

product_version<String>

Product Version.

uptime<String>

Uptime.

current_user_permission<String>

This property will show the permission type for current user.

platform<String>

Platform.

hypervisor_uptime<String>

Hypervisor Uptime.

product_build_number<String>

Product Build Number.

serial<String>

Serial Number.

host<String>

Host IP Address on which system is running, this will set for each client session only.

username<String>

User Name who is currently connected to the system.

current_tenant<String>

Current Tenant Name for logged-in user.

time_zone_offset<Integer>

Time zone offset in minutes (Example:-330).

build_number<String>

Build Number.

current_time_formatted<String>

Current Time (Formatted).

time_zone<String>

Server Time Zone.

product<String>

Product Name.

current_tenant_id<String>

Current Tenant Id for logged-in user.

Operations

NOTE: In request payload, Mandatory parameters are marked red and bold.

The following parameters can be used in the nitro request :

onerror   <String_value>

Use this parameter to set the onerror status for nitro request. Applicable only for bulk requests.
Default value: EXIT
Possible values = EXIT, CONTINUE
